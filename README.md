**Стек:** VLESS + XHTTP + Reality + Post-Quantum ML-KEM-768

## Как это работает

```
Клиент (v2rayTUN / Hiddify)
        │
        │  VLESS + XHTTP + Reality + ML-KEM-768
        │  (трафик выглядит как HTTPS к vkvideo.ru)
        ▼
┌──────────────────────────────┐
│       РФ-НОДА (входная)      │
│  VPS в России · порт 443     │
│  SNI-донор: vkvideo.ru       │
│                              │
│  Роутинг:                    │
│  *.ru / geoip:ru  → DIRECT   │
│  реклама          → BLOCK    │
│  остальное        → Евро     │
└──────────────┬───────────────┘
               │
               │  VLESS + TCP + Reality + ML-KEM-768
               │  (цепочка через зашифрованный туннель)
               ▼
┌──────────────────────────────┐
│      ЕВРО-НОДА (выходная)    │
│  VPS за рубежом · порт 443   │
│  SNI-донор: www.amd.com      │
└──────────────┬───────────────┘
               │
               ▼
           Интернет
```

### Почему два узла

Прямое подключение из России к зарубежному VPS детектируется и блокируется по паттерну трафика. РФ-нода выглядит как обычный HTTPS-сервер (vkvideo.ru), принимает соединения от клиента, и уже сама проксирует трафик через зашифрованную цепочку на выходной узел за рубежом.

### Post-Quantum ML-KEM-768

Xray-core поддерживает гибридное постквантовое шифрование **ML-KEM-768 + X25519**. Скрипт автоматически определяет поддержку установленной версии и активирует PQ, если доступно. Это защищает трафик от атак типа «перехвати сейчас, расшифруй потом» (harvest-now-decrypt-later).

### XHTTP + Reality

- **Reality** — маскировка под легитимный TLS-сервер (SNI-донор), без самоподписанных сертификатов
- **XHTTP** — транспорт поверх HTTP/2, обходит DPI глубже чем TCP

| | Евро-нода | РФ-нода |

| ОС | Ubuntu 20.04+ / Debian 11+ | Ubuntu 20.04+ / Debian 11+ |
| Доступ | root | root |
| Порт | 443/tcp открыт | 443/tcp открыт |

## Быстрый старт

Одной командой — скачать и запустить интерактивное меню:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/cmzmozg/xray-setup/main/setup-xray.sh)
```

Или сразу запустить нужную ноду:

```bash
# Евро-нода
bash <(curl -fsSL https://raw.githubusercontent.com/cmzmozg/xray-setup/main/setup-xray.sh) euro

# РФ-нода
bash <(curl -fsSL https://raw.githubusercontent.com/cmzmozg/xray-setup/main/setup-xray.sh) rf
```

## Порядок установки

### Шаг 1 — Евро-нода (зарубежный VPS)

```bash
./setup-xray.sh euro
# или выбрать пункт 1 в интерактивном меню
```

Скрипт выполнит:

1. Установку зависимостей (`curl`, `wget`, `ufw`, `openssl`, `python3` и др.)
2. Загрузку актуального **Xray-core** с GitHub Releases
3. Загрузку `geoip.dat` и `geosite.dat` (Loyalsoldier ruleset)
4. Генерацию ключей **X25519** (Reality) и UUID клиента
5. Активацию **Post-Quantum ML-KEM-768** (если поддерживается версией Xray)
6. Запись `/usr/local/etc/xray/config.json`
7. Базовую настройку UFW: deny all, SSH только с `ALLOWED_IPS`, порт 443 открыт
8. Запуск `xray` как systemd-службы

По завершении скрипт выводит **VLESS URI** для прямого подключения и сохраняет параметры в `euro-node.conf` рядом со скриптом.

### Шаг 2 — Перенос данных на РФ-сервер

Скопируйте скрипт и `euro-node.conf` в `/root/` на РФ-сервер:

```bash
scp setup-xray.sh euro-node.conf root@RF_SERVER_IP:/root/
```

> `euro-node.conf` должен находиться в `/root/` (или в той же папке откуда запускается скрипт). При запуске через `bash <(curl ...)` файл ищется автоматически в текущей директории.

### Шаг 3 — РФ-нода (российский VPS)

```bash
ssh root@RF_SERVER_IP
chmod +x setup-xray.sh
./setup-xray.sh rf
```

Скрипт автоматически найдёт `euro-node.conf` если он лежит в `/root/` (или в текущей папке) и загрузит параметры без лишних вопросов. Если файл не найден — запросит данные вручную.

По завершении будет выведен **VLESS URI для клиента** — добавьте его в v2rayTUN или Hiddify.

## Меню и аргументы

### Интерактивное меню

```
╔══════════════════════════════════════════════════╗
║     XRAY VLESS+XHTTP+Reality+PQ  INSTALLER      ║
╚══════════════════════════════════════════════════╝

  1) Настроить Евро-ноду   — установка выходного узла
  2) Настроить РФ-ноду     — установка входного узла-моста
  3) Показать статус        — состояние Xray и конфигурация
  4) Настройка портов       — закрытие опасных портов UFW
  0) Выход
```

### Аргументы командной строки

```bash
./setup-xray.sh           # интерактивное меню
./setup-xray.sh euro      # установить Евро-ноду
./setup-xray.sh rf        # установить РФ-ноду
./setup-xray.sh status    # статус текущей ноды
./setup-xray.sh ports     # управление портами UFW
```

## Настройка портов

Пункт `4) Настройка портов` — отдельный интерактивный раздел для управления UFW. Скрипт не закрывает порты автоматически при установке, чтобы избежать случайной потери доступа.

**Логика:**

1. Читает порт Xray из текущего конфига
2. Показывает список кандидатов на закрытие с текущим статусом в UFW
3. Если порт из списка совпадает с портом Xray — **пропускает** его с предупреждением
4. Предлагает добавить свои порты
5. Показывает итоговый список и запрашивает подтверждение `[y/N]`
6. Применяет правила и докладывает результат

Список портов по умолчанию: `80, 8080, 8443, 3389, 23, 21, 25, 110, 143`

## Диагностика

**Xray не запускается:**
```bash
journalctl -xe -u xray
cat /usr/local/etc/xray/config.json | python3 -m json.tool
```

**Порт 443 занят другим процессом:**
```bash
ss -tlnp | grep 443
fuser -k 443/tcp
systemctl restart xray
```

**Проверка Reality с клиентской машины:**
```bash
curl -v https://EURO_IP --resolve www.amd.com:443:EURO_IP
# Ожидаемый ответ: HTTP 200 OK
```

**Обновить geoip/geosite:**
```bash
wget -O /usr/local/share/xray/geoip.dat \
  https://github.com/Loyalsoldier/v2ray-rules-dat/releases/latest/download/geoip.dat
wget -O /usr/local/share/xray/geosite.dat \
  https://github.com/Loyalsoldier/v2ray-rules-dat/releases/latest/download/geosite.dat
systemctl restart xray
```

## Безопасность

- **SSH** ограничен по IP на обеих нодах — доступ только с адресов в массиве `ALLOWED_IPS` в начале скрипта. Отредактируйте список **до запуска**.
- **UFW**: `deny incoming`, `allow outgoing`, `deny forward` по умолчанию. Закрытие опасных портов — вручную через пункт `4)`.
- **Reality** не требует сертификатов, трафик неотличим от легитимного TLS к SNI-донору.
- **PQ ML-KEM-768** защищает от атак на будущих квантовых компьютерах.

```bash
# Перед запуском отредактируй список разрешённых IP:
ALLOWED_IPS=("ВАШ_IP_1" "ВАШ_IP_2")
```
